# Как создать первого админа, после установки
kubectl exec -n teleport -ti teleport-auth-5f75cfbfb7-dnf6w -- tctl users add admin --roles=editor,access --logins=root,ubuntu,admin

1. --roles
   1. Это уже именно роли пользователя
   2. Изначально, их достпуно немного (около 10ти штук)
   3. Можно создаь Role - самостоятельно. Но это прям ... - СЛОЖНО
2. --logins === user_traits
   1. Как понял - это какие-то дополнительные флаги у пользователя
   2. в данном примере: мы указали что пользователь может логиниться на сервера по SSH только под этими logins
   3. Когда будет установлен SSH агент и пользователь зайдет в WEB-UI - только указанные тут logins будут доступны для коннекта
3. В результате команды = будет ссылка
   1. Переходим по этой сслыке
   2. задаем пароль
   3. Включаем 2FA
   4. Мы внутри

# как установить ssh-agent
1. Получить токен
   1. kubectl exec -it -n teleport teleport-auth-7c9dc6c54d-prwhb -- tctl tokens add --type=node --ttl=24h --format=json
   2. Что в результате (сноска_1)
   3. поле --type=node === Это Roles (Это фиксированный список: app, node, db, kube, bot, WindowsDesktop, Discovery)
2. при инициализации

# Какая структура
- user = тот кто зашел в web-ui (пока-что назовем так)
- roles = что ему вообще можно делать
  - Ты можешь зайти на сервер по SSH
- traits = какие значения ты можешь использовать
  - Ты можешь зайти на сервер по SSH (на основе твоей роли) использую logins (aaa,bbb,ccc)
